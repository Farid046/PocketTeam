# /product-brief — Product Brief

Create a structured brief that the Planner can directly use for implementation planning.

## Brief Template

```markdown
## Product Brief: [Feature Name]

### Problem Statement
What problem does this solve? Who experiences it? How severe is it?

### Proposed Solution
High-level description of what we're building.

### Success Criteria
- [ ] Metric 1: [measurable outcome]
- [ ] Metric 2: [measurable outcome]

### Scope
**In scope:** [list]
**Out of scope:** [list]

### Risks & Open Questions
1. [Risk/question + mitigation]

### Priority
- Impact: High/Medium/Low
- Effort: High/Medium/Low
```

## Process
1. Fill in each section based on demand validation results
2. Be specific about scope boundaries
3. Success criteria must be measurable
4. Hand completed brief to COO for routing to Planner
